# Experiment Result Comparator

Takes structured experiment results and produces publication-ready comparison tables in LaTeX (booktabs format) and Markdown, along with rankings and trade-off analysis text.

## When to Use

- User has experiment results (model names + metric values) and needs a paper table
- Comparing 2+ models across multiple metrics (F1, accuracy, BLEU, latency, etc.)
- Needs trade-off analysis text (e.g. accuracy vs. speed)
- Generating ablation study tables

## Input Format

Prepare a JSON file with this structure:

```json
{
  "experiments": [
    {"name": "BERT-base",     "metrics": {"F1": 88.5, "Precision": 87.2, "Latency_ms": 120}},
    {"name": "RoBERTa-large", "metrics": {"F1": 91.2, "Precision": 90.8, "Latency_ms": 280}},
    {"name": "DistilBERT",    "metrics": {"F1": 85.3, "Precision": 84.1, "Latency_ms": 65}}
  ],
  "primary_metric": "F1",
  "higher_is_better": {"F1": true, "Precision": true, "Latency_ms": false},
  "output_format": "all",
  "caption": "Results on CoNLL-2003 NER test set.",
  "label": "tab:ner_results"
}
```

**Field notes:**
- `higher_is_better`: omit to default all metrics to higher=better. Set to `false` for latency, perplexity, error rate, etc.
- `output_format`: `"all"` returns LaTeX + Markdown + analysis. Use `"latex"` or `"markdown"` alone for just the table.
- `caption` and `label` are used only for the LaTeX table.

## Step-by-Step Instructions

1. **Ask the user** for their experiment results if not already provided. Minimum needed: model names and numeric metric values.

2. **Create the input JSON file** at a temporary path, e.g. `/tmp/experiments.json`.

3. **Run the comparison script**:

   ```bash
   python skills/experiment-result-comparator/run.py --input /tmp/experiments.json
   ```

4. **Parse the output JSON** and present to the user:
   - Show the **Markdown table** inline in the conversation for readability
   - Provide the **LaTeX table** as a code block they can paste into their paper
   - Summarize the **trade-off analysis** paragraph
   - List the **ranking** by primary metric
   - Highlight any notable findings from `conclusions`

5. If the user wants an ablation table (rows = model variants with component checkmarks), see `docs/ablation-tables.md`.

## LaTeX Table Format

The generated LaTeX uses `booktabs` style:
- Best value per metric → **bold** (`\textbf{...}`)
- Second-best per metric → underlined (`\underline{...}`)
- Sorted by primary metric (descending by default)

Requires `\usepackage{booktabs}` in the LaTeX preamble.

## Notes

- Trade-off analysis is generated by an LLM and takes ~3 seconds. Set `"include_tradeoff": false` to skip.
- For ablation studies with component checkmarks, see `docs/ablation-tables.md`.
- All numeric values are formatted to 2 decimal places by default. Override with `"decimals": 1`.
